# CLAUDE.md

このファイルは、Claude Code がこのリポジトリで作業する際のコンテキスト・前提・運用ルールをまとめたもの。

## プロジェクト概要

**overdue-shepherd** は、Apple 純正「リマインダー」アプリの期限切れタスクを毎朝「世話する」 Scriptable スクリプト群。前身は [ticktick-tag-cleaner](https://github.com/khidaka/ticktick-tag-cleaner) で、TickTick 用ロジックを Apple リマインダー向けに移植したもの。

## 実行環境

- iPhone (iOS 16+) + Scriptable アプリ + iOS ショートカット (オートメーション)
- すべて **ローカル実行** — OAuth / 外部 API / Keychain は不要
- Scriptable の組み込み `Reminder` クラス (EventKit ラッパー) 経由でリマインダー DB を直接操作

## ファイル構成

| ファイル | 役割 |
|---|---|
| `OverdueShepherd.js` | 本体。3 ケースを処理し優先度を付ける |
| `OverdueShepherdDiagnostic.js` | 副作用なしの診断 (ドライラン) |
| `OverdueShepherdMigrateTag.js` | 単発移行用。旧 `#postponed_Nd` → 新 `(N日遅延)` 置換のみ。期日/優先度は触らない |
| `README.md` | エンドユーザー向けセットアップ手順 |
| `CLAUDE.md` | このファイル (Claude Code 用コンテキスト) |

## 処理ロジック

全リスト横断で `Reminder.allIncomplete()` を取り、未完了タスクごとに以下を判定:

| ケース | 条件 | アクション |
|---|---|---|
| A | 期日 ≥ 今日 0:00 + `(N日遅延)` あり | タグ除去 + `priority = 0` |
| B | 期日 < 今日 0:00 + `(N日遅延)` あり | `N→N+1`、期日を今日に (時刻保持)、優先度を更新 |
| C | 期日 < 今日 0:00 + タグなし | `(1日遅延)` を末尾付与、期日を今日に (時刻保持)、優先度を低 |

完了済み (`isCompleted`) と期日なしはスキップ。

### 優先度マッピング (スマートリスト連携用)

EventKit の priority 値 (0=なし / 1=高 / 5=中 / 9=低) を使う:

| 遅延日数 N | 優先度 |
|---|---|
| 1d | 9 (低) |
| 2d | 5 (中) |
| 3d 以上 | 1 (高) |

## 重要な設計判断・前提

### 1. `(N日遅延)` は「Reminders タグ」ではなく「タイトル中の素のテキスト」

- EventKit は Apple リマインダーのネイティブタグ (UI から付ける `#hashtag`) を**公開していない**
- そのため Scriptable からタグの読み書きはできない
- 本スクリプトは `r.title` を文字列として正規表現マッチ・置換するだけ (`TAG_RE = /\((\d+)日遅延\)/`)
- ユーザーには「リマインダー UI のタグ機能とは別物として運用する」よう README で案内済み

#### 1a. なぜ `#postponed_Nd` から `(N日遅延)` に変えたか (v1.1.0, 2026-04-29)

- iOS ショートカットの通知トリガで Siri がリマインダー本文を読み上げる運用になっており、`#postponed_1d` は「ハッシュ ポストポンド アンダースコア…」と記号読みされて聞き取りに失敗していた
- 日本語表記 `(N日遅延)` に変更し、半角括弧で前後を区切ってタイトル本文と分離 (全角括弧ではない点に注意)
- 移行は単発の `OverdueShepherdMigrateTag.js` を 1 回実行して完了。本体には旧タグ互換コードは入れない方針 (シンプルさ優先)

### 2. 期日変更時は時刻成分を保持する

ユーザー要望: 期日を「今日」に繰り越す際、元の時刻 (例: 14:00) は維持する。
実装は `shiftDateKeepingTime()` で、`setFullYear(year, month, date)` のみ呼んで時/分/秒は元の Date から引き継ぐ。比較用の「今日 0:00」とは別に切り出してある。

### 3. 繰り返しタスクも同じロジックで処理する

ユーザー合意済み。`dueDate` を書き換えると Apple 側の繰り返し計算に影響する可能性があるが、運用上問題なしとして許容。

### 4. スマートリスト連携は priority 経由で行う

- スマートリストのフィルタは「タグ」「期日」「優先度」「フラグ」「リスト」など限られた属性のみ
- タイトル文字列でのフィルタは不可
- `isFlagged` (旗) はユーザーが「当日フォーカスするタスク」用に使う運用。前日分の旗が翌日に残ると邪魔なので毎朝全解除したいが、**Scriptable の Reminder クラスは flag 属性を公開していない** (タグと同じく EventKit ブリッジ漏れ)。`r.isFlagged` は常に undefined になるため Scriptable 側からは実装不可能 (v1.2.0 で試みて撤回)
- 旗の朝リセットは iOS ショートカット側で実装する: 「リマインダーを検索 (フラグ付き)」→「各項目を繰り返す」→「繰り返し項目のフラグ付きをいいえに設定」→「繰り返しの終了」。「フラグ付きを設定」アクションは単一入力しか受け付けないため Repeat with Each ループが必須。値トークンは「毎回尋ねる」ではなくリテラル指定すること (オートメーション中にダイアログが出る)。手順は README 参照
- 優先度のみ使う (これがユーザーの明示的な指示)

### 5. ユーザーは GitHub オーナー `khidaka`、リポジトリ `overdue-shepherd`

- リモート: `https://github.com/khidaka/overdue-shepherd`
- main ブランチに直接 push する運用

### 6. スクリプト先頭にバージョンヘッダを入れる

ユーザー要望: Scriptable に貼り直したときどの版か分かるよう、スクリプト先頭コメントに `Version: x.y.z (YYYY-MM-DD)` を必ず記載する。本体・診断スクリプトは互換ロジックを共有するため版を同期させる。単発の `OverdueShepherdMigrateTag.js` は独立した版番号で OK。

### 7. Siri 読み上げを壊さないこと (v1.1.0 以降の制約)

- 通知トリガで Siri がリマインダー本文を読み上げる運用なので、タイトルに付与する文字列は**自然な日本語として発話できる形**を維持する
- ハッシュタグ・アンダースコア・英略語などの記号読みされる文字列を、タイトルへ機械的に書き込むのは避ける
- 将来タグ仕様を変える場合も「Siri が違和感なく読めるか」を判断基準に含める

## 開発フロー

1. ファイルを編集
2. 必要ならバージョンヘッダを更新
3. `git add -A && git commit -m "..." && git push` で main に push
4. ユーザーが iPhone Scriptable に手動でコピペし直して反映

### 移行スクリプトの扱い

- タグ仕様や保存形式を破壊的変更する場合は、本体に互換コードを混ぜず、`OverdueShepherd<目的>.js` という単発スクリプトを別ファイルで作って 1 回実行する方針 (v1.1.0 のタグ移行 = `OverdueShepherdMigrateTag.js` がリファレンス実装)
- 単発スクリプトは「再実行しても no-op」になるよう冪等に書く
- 副作用ありのスクリプトは件数とサンプル変換結果をログ + Alert に出して結果を確認できるようにする

## 運用 (ユーザーがやっていること)

- 毎朝の自動実行は iOS ショートカット + 個人用オートメーション (時刻トリガー) で構成済み
- 「実行の前に尋ねる」OFF + 通知お好みで完全自動化
- 集中モード/おやすみモードは実行を止めないため対策不要 (通知のみ抑制される)

## 既知の注意点

- リマインダーの「タグ機能」と本スクリプトの `(N日遅延)` は完全に別物。混在運用に注意
- 繰り返しリマインダーへの `dueDate` 書き換えによる挙動は実機検証で許容範囲と確認済み
- Scriptable のコンソールはエディタ画面下端から上スワイプで表示。見つけにくい場合は QuickLook で代替可能
