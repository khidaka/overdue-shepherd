# CLAUDE.md

このファイルは、Claude Code がこのリポジトリで作業する際のコンテキスト・前提・運用ルールをまとめたもの。

## プロジェクト概要

**overdue-shepherd** は、Apple 純正「リマインダー」アプリの期限切れタスクを毎朝「世話する」 Scriptable スクリプト群。前身は [ticktick-tag-cleaner](https://github.com/khidaka/ticktick-tag-cleaner) で、TickTick 用ロジックを Apple リマインダー向けに移植したもの。

## 実行環境

- iPhone (iOS 16+) + Scriptable アプリ + iOS ショートカット (オートメーション)
- すべて **ローカル実行** — OAuth / 外部 API / Keychain は不要
- Scriptable の組み込み `Reminder` クラス (EventKit ラッパー) 経由でリマインダー DB を直接操作

## 処理ロジック

全リスト横断で `Reminder.allIncomplete()` を取り、未完了タスクごとに以下を判定:

| ケース | 条件 | アクション |
|---|---|---|
| A | 期日 ≥ 今日 0:00 + `#postponed_Nd` あり | タグ除去 + `priority = 0` |
| B | 期日 < 今日 0:00 + `#postponed_Nd` あり | `N→N+1`、期日を今日に (時刻保持)、優先度を更新 |
| C | 期日 < 今日 0:00 + タグなし | `#postponed_1d` を末尾付与、期日を今日に (時刻保持)、優先度を低 |

完了済み (`isCompleted`) と期日なしはスキップ。

### 優先度マッピング (スマートリスト連携用)

EventKit の priority 値 (0=なし / 1=高 / 5=中 / 9=低) を使う:

| 遅延日数 N | 優先度 |
|---|---|
| 1d | 9 (低) |
| 2d | 5 (中) |
| 3d 以上 | 1 (高) |

## 重要な設計判断・前提

### 1. `#postponed_Nd` は「Reminders タグ」ではなく「タイトル中の素のテキスト」

- EventKit は Apple リマインダーのネイティブタグ (UI から付ける `#hashtag`) を**公開していない**
- そのため Scriptable からタグの読み書きはできない
- 本スクリプトは `r.title` を文字列として正規表現マッチ・置換するだけ
- ユーザーには「リマインダー UI のタグ機能とは別物として運用する」よう README で案内済み
- 検証時に既存の UI タグを削除しないと、二重に `#postponed_1d` が付与される問題が起きた経緯あり

### 2. 期日変更時は時刻成分を保持する

ユーザー要望: 期日を「今日」に繰り越す際、元の時刻 (例: 14:00) は維持する。
実装は `shiftDateKeepingTime()` で、`setFullYear(year, month, date)` のみ呼んで時/分/秒は元の Date から引き継ぐ。比較用の「今日 0:00」とは別に切り出してある。

### 3. 繰り返しタスクも同じロジックで処理する

ユーザー合意済み。`dueDate` を書き換えると Apple 側の繰り返し計算に影響する可能性があるが、運用上問題なしとして許容。

### 4. スマートリスト連携は priority 経由で行う

- スマートリストのフィルタは「タグ」「期日」「優先度」「フラグ」「リスト」など限られた属性のみ
- タイトル文字列でのフィルタは不可
- `isFlagged` はユーザーが「進行中タスク」用に使っているため**触らない**
- 優先度のみ使う (これがユーザーの明示的な指示)

### 5. ユーザーは GitHub オーナー `khidaka`、リポジトリ `overdue-shepherd`

- リモート: `https://github.com/khidaka/overdue-shepherd`
- main ブランチに直接 push する運用

### 6. スクリプト先頭にバージョンヘッダを入れる

ユーザー要望: Scriptable に貼り直したときどの版か分かるよう、スクリプト先頭コメントに `Version: x.y.z (YYYY-MM-DD)` を必ず記載する。両スクリプトで同期させる。

## 開発フロー

1. ファイルを編集
2. 必要ならバージョンヘッダを更新
3. `git add -A && git commit -m "..." && git push` で main に push
4. ユーザーが iPhone Scriptable に手動でコピペし直して反映

## 運用 (ユーザーがやっていること)

- 毎朝の自動実行は iOS ショートカット + 個人用オートメーション (時刻トリガー) で構成済み
- 「実行の前に尋ねる」OFF + 通知お好みで完全自動化
- 集中モード/おやすみモードは実行を止めないため対策不要 (通知のみ抑制される)

## 既知の注意点

- リマインダーの「タグ機能」と本スクリプトの `#postponed_Nd` は完全に別物。混在運用に注意
- 繰り返しリマインダーへの `dueDate` 書き換えによる挙動は実機検証で許容範囲と確認済み
- Scriptable のコンソールはエディタ画面下端から上スワイプで表示。見つけにくい場合は QuickLook で代替可能
