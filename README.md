# overdue-shepherd

Apple 純正「リマインダー」アプリの期限切れタスクを毎朝「世話する」 Scriptable スクリプト。

iPhone 上で毎朝自動実行し、期限切れタスクに `#postponed_Nd` タグを付けて遅延日数を可視化しつつ期日を今日に繰り越す。前身の [ticktick-tag-cleaner](https://github.com/khidaka/ticktick-tag-cleaner) を Apple リマインダー向けに移植したもの。

## 仕組み

全リスト横断で未完了リマインダーを走査し、3 ケースで処理する:

| ケース | 条件 | アクション |
|---|---|---|
| A | 期日が今日以降 + `#postponed_Nd` タグあり | タグ除去 (リセット) + 優先度をなしに |
| B | 期日が過去 + `#postponed_Nd` タグあり | `N` を `N+1` に更新し、期日を今日へ + 優先度を更新 |
| C | 期日が過去 + タグなし | `#postponed_1d` を付与し、期日を今日へ + 優先度を低に |

### 優先度のマッピング (スマートリスト連携用)

スマートリストはタイトル内のテキストをフィルタできないため、遅延日数を **優先度** に反映してスマートリストから引けるようにしている。

| 遅延日数 (N) | 優先度 |
|---|---|
| 1d | 低 |
| 2d | 中 |
| 3d 以上 | 高 |

リマインダーアプリで以下のスマートリストを作ると活用しやすい:
- 「優先度: 高」 → 3 日以上滞留している重度の遅延タスク
- 「優先度: 中」 → 2 日遅延中
- 「優先度: 低」 → 1 日遅延中
- 「優先度なし」 → 通常タスク

- 完了済みタスクは触らない
- 期日なしタスクはスキップ
- 期日を今日に変更する際、**元の時刻成分は保持** (年月日のみ書き換え)
- `#postponed_Nd` タグが Apple リマインダー上に未登録の `N` 値でも、タイトルへの hashtag 書き込みで自動的に新規タグとして登録される

## 必要なもの

- iPhone (iOS 16 以降推奨。タグ機能のため iOS 15+ 必須)
- [Scriptable](https://scriptable.app/) (無料)
- 「ショートカット」アプリ (標準搭載)

TickTick のような OAuth / Developer 登録 / Premium 契約は不要。すべてローカルで完結する。

## ファイル

| ファイル | 役割 |
|---|---|
| `OverdueShepherd.js` | メインスクリプト |
| `OverdueShepherdDiagnostic.js` | 診断用 (リマインダー側を変更せず判定結果のみログ出力) |

## セットアップ

1. App Store から Scriptable をインストール
2. `OverdueShepherd.js` の中身を Scriptable に新規スクリプトとして貼り付ける (ファイル名はスクリプト名と一致させる)
3. 同様に `OverdueShepherdDiagnostic.js` も追加 (任意。挙動確認に使用)
4. 初回実行時に表示されるリマインダーへのアクセス許可ダイアログを承認
5. 「ショートカット」アプリで新規ショートカットを作成:
   - アクション: 「**スクリプトを実行**」 (Scriptable) → スクリプト名 `OverdueShepherd` を選択
6. 「オートメーション」→「個人用オートメーションを作成」→「時刻」で毎朝の希望時刻 (例: 6:00) を設定し、上記ショートカットを実行するよう設定
   - 「実行の前に尋ねる」を **オフ** にすると完全自動化される

## 動作確認手順

1. リマインダーアプリでテスト用リスト (例: `test-shepherd`) を作成し、以下を投入:
   - 期日 = 昨日 09:30、タグなし → ケース C 期待 (今日 09:30 になり `#postponed_1d` が付く)
   - 期日 = 昨日 14:00、タイトル末尾に `#postponed_2d` → ケース B 期待 (今日 14:00 / `#postponed_3d` に)
   - 期日 = 明日、`#postponed_1d` 付き → ケース A 期待 (期日そのまま、タグ除去)
   - 期日 = 昨日、完了済み → 変更なし
   - 期日 = 昨日 (時刻なし、終日) → 期日のみ今日に
2. Scriptable で `OverdueShepherdDiagnostic` を実行し、判定が想定どおりかコンソールで確認
3. `OverdueShepherd` を手動実行し、リマインダーアプリ側の表示を確認 (時刻が保持されていること)
4. ショートカットからの実行 → オートメーションでの自動実行を順に確認

## 重要: リマインダーの「タグ」とは別物として扱う

Apple リマインダーで UI から付けた `#hashtag` は、内部的にはタイトル本文とは別フィールドに格納されており、Scriptable (EventKit) の API からは読めない。本スクリプトは `r.title` の文字列だけを見て判定・書き換えするため、**リマインダーアプリ純正のタグ機能とは別物として運用する**:

- 本スクリプトが付与する `#postponed_Nd` は「タイトル末尾に書かれたただのテキスト」である (見た目はタグだが、リマインダーの正式なタグではない)
- 既存のリマインダーで UI 経由でタグとして付けた `#postponed_*` がある場合は、初回運用前に手動で削除しておくこと (そのままだとスクリプトからは「タグなし」と判定され、二重に `#postponed_1d` が追加される)
- 入力中にリマインダーが「タグ化」を提案してきたら無視してプレーンテキストとして確定する

## その他の注意

- 繰り返し (recurring) リマインダーも同じロジックで処理される。`dueDate` を書き換えると Apple 側の繰り返し計算に影響する可能性があるため、運用前に挙動を確認すること
- スクリプトは全リスト横断で動作する。対象を絞りたい場合は `Reminder.allIncomplete()` の結果を `r.calendar.title` でフィルタするコードを追加する
